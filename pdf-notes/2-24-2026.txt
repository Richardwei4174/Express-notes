Header include the following:

Cookies

Http cookies - small pieces of data that a web server sends to the browser. The browser can store the cookie. The server can tell who’s cookie it belongs to and  dynamically render based on the cookie. It helps the browser to identify the user.

Install cookie parser to parse through cookie(in the terminal):
npm i cookie-parser

To use it we need the header:
	import cookieParser from "cookie-parser";

app.use(cookieParser()); // to enable cookieParser middleware

When you send a cookie, the server would check for the cookie and if the cookie is the correct value and exists then it will send the correct information back to the users.

Here’s an example:
router.get("/api/products",(request, response)=>{
  console.log(request.headers.cookie);
  console.log(request.cookies);
  if (request.cookies.hello && request.cookies.hello === 'world'){
    return response.send([{id: 123, name: 'cucumber', price: "12.99"},]);
  }
  return response.send({msg: 'Sorry, you need the correct cookie'});
});
// check to see if the cookie with the key hello exists and see if the value is ‘world’. 
// if it is than we would send the data with the cucumber components but 
// if it doesn’t exists then we would send the sorry message

Validation

Sometimes the data you expect is not the data that you received. Validation is important for security in case someone tries to bypass it by sending a request outside of the client front end side. Ensure that the information we are receiving is the information that we are expecting. 

To download in vscode type the following in the terminal:
npm i express-validator
To use it we would need to import the following:
	import {query, validationResult} from "express-validator"; // to help validate
Those imported functions are middle ware which means we are calling them before we call our request functions 

query(‘filter’) object has many builtin validator like query(‘filter’).isString() to check if the input is a string

It can also work as a chain to validate multiple things at once such as
	query(‘filter’).isString().notEmpty()
So it is a string and also not empty

Key thing to note: query() would tell you if there’s a problem but it won’t handle it for you so you’d need to call validationResult() to help handle it for you

Query Params
‘?’ at the end of a website denotes a query string. It’s for key value pairs. We can use ‘&’ to add more of those pairs.

Essentially it means we get some kind of key value pair to the server side and we can do something about it. For instance we can do 
http://localhost:3000/?filter=username&value=an
Which would create a json request {filter: ‘username’, value:’an’} where we can decide what to do with those values.

Let’s say using the example above, we can check if filter and value “exist” as in they are on the website route (in this case yes) and than filter our our premade list of username.

Say we have a list of username and displayName as such:

const mockUsers = [
  {id: 1, username: "anson", displayName: "Anson"},
  {id: 2, username: "bobl", displayName: "Bob2"},
  {id: 3, username: "demonslayer3000", displayName: "Bob3"},
];

// we can filter out our list based on the value of “value” which is “an” in this example. It would go through the list and return back the list with “an” in the username

app.get('/', (request, response)=>{
  console.log(request.query);
  const{ query: {filter, value} } = request;
  
  if (filter && value) return response.send(
    mockUsers.filter( (user) => // no {} else it won't work
      // filter al the filter object that matches value
      user[filter].includes(value)
    )
  )
  // return normal is there's no filter AND value in the route
  return response.send(mockUsers);

});
